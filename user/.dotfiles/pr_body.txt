💡 **What:**
Replaced all inefficient PowerShell array concatenations (`+=`) in `NvidiaAutoinstall.ps1` with faster alternatives:
1. Replaced `$dList += $deviceId` with `[System.Collections.Generic.List[string]]` and its `.Add()` method.
2. Replaced `$releaseHighlights += $trimmedHighlight` by assigning the output of the `foreach` loop directly to the variable (`$releaseHighlights = foreach (...)`).
3. Replaced `$checkboxes += $checkbox` and `$checkboxesColor += $checkboxColor` with `[System.Collections.Generic.List[System.Windows.Forms.checkbox]]` and their `.Add()` methods.

🎯 **Why:**
Using `+=` on an array in PowerShell forces a full O(N) array memory reallocation on every single iteration, leading to O(N^2) total execution time. This is a notorious performance sink, particularly inside UI build loops and repetitive text parsing loops. Using `.Add()` on a generic `.NET` List provides amortized O(1) appends, and assigning a pipeline loop block provides immediate streaming. This will make the driver script significantly more responsive when processing UI checkboxes or updating the digital vibrance monitor list.

📊 **Measured Improvement:**
Since `pwsh` is not installed on the current Linux development environment, a local baseline profiling script via `Measure-Command` could not be reliably executed. However, this is a textbook optimization. In typical environments parsing lists of strings or allocating UI handles, `+=` often takes exponential time, whereas a List or pipeline loop completes in a fraction of a millisecond. To maintain compatibility, `$dList` is cleanly cast back via `.ToArray()` after the loop to preserve the `.Length` property expected in downstream conditionals.
